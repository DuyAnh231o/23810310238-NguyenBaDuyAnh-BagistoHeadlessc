So sánh lưu lượng dữ liệu:
Khi dùng REST API lấy tất cả dữ liệu, payload trả về thường lớn vì chứa mọi trường, kể cả không cần, dẫn đến tốn băng thông. GraphQL cho phép chỉ định trường cần thiết, nên payload nhỏ hơn, hiệu quả hơn và giảm tải mạng.
Thay đổi giá sản phẩm qua Headless API:
Em sẽ dùng Mutation vì GraphQL phân biệt Query chỉ để đọc dữ liệu, còn Mutation để thay đổi hoặc cập nhật dữ liệu trên server, như cập nhật giá sản phẩm.
